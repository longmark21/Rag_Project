# RAG 系统问题修复说明

## 问题总结

### 1. 聊天功能失败 - 数据库问题
**错误信息**：`org.postgresql.util.PSQLException: FATAL: database "rag_vector" does not exist`

**原因**：PostgreSQL 向量数据库 `rag_vector` 不存在，导致无法检索文档内容。

**解决方案**：
1. 在 PostgreSQL 服务器 (192.168.31.29:5434) 上创建数据库
2. 执行 `init-vector-db.sql` 脚本创建必要的表和索引

**执行步骤**：
```bash
# 连接到 PostgreSQL
psql -h 192.168.31.29 -p 5434 -U postgres

# 创建数据库
CREATE DATABASE rag_vector;

# 连接到新数据库
\c rag_vector;

# 执行初始化脚本
\i init-vector-db.sql
```

或者手动执行 SQL 脚本中的内容。

### 2. 聊天功能失败 - API 调用问题
**错误信息**：`data: 抱歉，回答您的问题时出现错误`

**原因**：使用的是模拟的 ChatClient，不是真实的 Kimi API。

**解决方案**：
1. 创建了 `KimiService.java` - 真实的 Kimi API 调用服务
2. 修改了 `ChatClient.java` - 使用 KimiService 进行真实调用
3. 修改了 `ChatModelConfig.java` - 注入真实的 KimiService

**API Key 配置**：
已在 `application.yml` 中配置：
```yaml
spring:
  ai:
    moonshot:
      api-key: abc
      base-url: https://api.moonshot.cn/v1
      chat:
        options:
          model: moonshot-k2-thinking
          temperature: 0.3
```

### 3. 前端字段名不匹配
**问题**：前端发送 `message`，后端期望 `question`

**解决方案**：
- 修改 `api.ts`：将 `ChatRequest` 接口的 `message` 字段改为 `question`
- 修改 `store.ts`：调用 `streamChat` 时使用 `question: message`

### 4. 文档列表不显示
**问题**：后端返回格式是 `{ success: true, data: [...] }`，但前端直接返回响应

**解决方案**：
- 修改 `api.ts`：更新 `getDocuments` 函数的返回类型
- 修改 `store.ts`：正确处理 `response.success` 和 `response.data`

### 5. 文件上传后不显示文档
**问题**：后端返回的格式与前端期望的格式不匹配

**解决方案**：
- 修改 `api.ts`：扩展 `UploadResponse` 接口
- 修改 `store.ts`：将后端响应转换为前端 Document 格式

## 错误信息说明

### Kimi API 常见错误

1. **401 Unauthorized**
   - 原因：API Key 无效或过期
   - 解决：检查 `application.yml` 中的 `spring.ai.moonshot.api-key`

2. **403 Forbidden**
   - 原因：API Key 余额不足或权限不足
   - 解决：充值 Kimi 账户或检查账户状态

3. **429 Too Many Requests**
   - 原因：请求过于频繁
   - 解决：稍后重试或降低请求频率

4. **500 Internal Server Error**
   - 原因：Kimi 服务器内部错误
   - 解决：稍后重试

- **AI 模型**：Kimi (Moonshot) - moonshot-k2-thinking
- **向量模型**：通义千问 - text-embedding-v2 (1536 维度)
